# Do Hardware ao Sistema

Antes de abrir um terminal ou digitar o primeiro comando, vale entender o terreno. Este capítulo constrói a base conceitual que sustenta tudo o que vem depois.

Começamos pelos *dispositivos*: desktops, smartphones, servidores e sistemas embarcados, cada um com sua arquitetura, propósito e restrições de hardware.

Em seguida, vemos o que é *software* e como ele se organiza, da distinção entre software de sistema e software de aplicação até os conceitos de código aberto e software livre que estão no coração do Linux.

Depois, exploramos os principais *sistemas operacionais*, do Windows ao macOS, passando pelo Linux e pelos sistemas móveis, com atenção especial aos SOs voltados para servidores.

Com essa base, entramos nas *distribuições Linux*: o que são, por que existem tantas e como se organizam nas três grandes famílias, Debian, Red Hat e Arch, além de outras linhagens relevantes.

Por fim, olhamos para a camada visual do Linux. Primeiro os *ambientes de desktop*, como GNOME, KDE Plasma, XFCE e Cinnamon, cada um com uma filosofia distinta de uso. Depois, para quem quiser ir além, os *gerenciadores de janelas em mosaico*, uma categoria de interface radicalmente diferente, onde o teclado substitui o mouse e cada pixel da tela tem um propósito.
